# certinfo - инспектор ГОСТ-сертификата

Разбирает сертификат X.509 с ГОСТ-алгоритмами и выводит данные в человекочитаемом формате: Subject/Issuer с расшифровкой российских OID-ов (ИНН, ОГРН, СНИЛС, ИНН ЮЛ и др.), срок и статус, алгоритм (2012-256 / 2012-512 / 2001), серийный номер, отпечатки и расширения. Российские идентификаторы `openssl x509 -text` показывает сырыми числами (`1.2.643.100.4` и т.п.), либо вовсе не расшифровывает.

Работает без КриптоПро и без gost-engine: разбор структуры `crypto/x509` (ГОСТ-серты она читает; открытый ключ помечает «unknown», но всё остальное доступно), OID-ы алгоритмов достаются разбором ASN.1. Единственнаязависимость — [gogost](https://github.com/ddulesov/gogost) (GPL v3) для ГОСТ-отпечатка Streebog-256.

## Использование

```bash
make certinfo-build                                # собрать образ (однократно)
make certinfo FILE=data/certs/valid.cer            # текстовый вывод
make certinfo FILE=data/certs/dskogueg.cer JSON=1  # вывод в JSON-формате
```

На вход принимаются PEM (`.cer`/`.pem`) или DER. Через прямой запуск можно читать и со stdin:

```bash
docker run --rm -i crypto-amateur-certinfo - < cert.der
```

## Что показывает

- **Субъект / Издатель** — все атрибуты DN, российские OID расшифрованы.
- **Срок действия** — с/по и статус (`действует` / `истёк` / `ещё не вступил`),  для действующего сертификата - сколько дней осталось.
- **Алгоритм** — открытого ключа и подписи (с OID и разрядностью).
- **Серийный номер** (hex и dec), версия.
- **Отпечатки** — SHA-1 (совпадает с thumbprint в КриптоПро), SHA-256, Streebog-256.
- **Использование ключа** (KeyUsage / EKU), **расширения**: ограничения (CA),  идентификаторы ключей, точки CRL, OCSP/AIA, политики (классы КС1/КС2/КС3),  Средства ЭП владельца и издателя, прочие extension по OID.

## Сборка нативного бинарника (если есть Go)

```bash
cd certinfo && go build -mod=vendor -o certinfo .
./certinfo ../data/certs/valid.cer
```
