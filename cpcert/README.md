# cpcert - конвертер контейнеров КриптоПро в PEM

Скорее всего библиотеки, с которыми вы работаете (например, [ekapusta/oauth2-esia](https://github.com/ekapusta/oauth2-esia)) работают не с pfx, а с вполне привычной парой (сертификат + приватный ключ) - возникает ситуация, что вам скидывают pfx или выгрузку [ключей](../docs/keys.md), но что с ними делать непонятно. Как достать сам контейнер смотрите в [этом разделе](../docs/keys.md) - полученную директорию скопируйте в `data/certs`. И уже с полученными файлами этот конвертер придет к вам на помощь.

Это реализация на Golang, вдохновлённая
[get-cpcert](https://github.com/kov-serg/get-cpcert) и использующая [gogost](https://github.com/ddulesov/gogost) (GPL v3).

Она достаёт сертификат и приватный ключ из контейнера КриптоПро и выводит их в обычном PEM-формате, пригодном для OpenSSL. Поддержка: ГОСТ-2001 / ГОСТ-2012-256 / ГОСТ-2012-512

## Почему из контейнера `.000`, а не из `.pfx`

Казалось бы, проще экспортнуть `.pfx` (PKCS#12) и вытащить ключ обычным `openssl pkcs12`. Но с экспортом КриптоПро это не работает для приватного ключа:

- **Сертификат** в `.pfx` зашифрован стандартно (`pbeWithSHA1And3-KeyTripleDES-CBC`)
  достаётся без проблем: `openssl pkcs12 -in file.pfx -nokeys -clcerts -out cert.cer`.
- **Приватный ключ** КриптоПро шифрует проприетарным GOST-PBE (OID `1.2.840.113549.1.12.1.80`), которого нет в таблице алгоритмов OpenSSL даже с `-engine gost` и openssl падает с `unknown pbe algorithm` / `Error outputting keys`.

Поэтому ключ берём напрямую из контейнера и далее разбор ASN.1, KDF на ГОСТ-хэше, расшифровка GOST 28147, снятие маски.

## Использование

Файлы создаются раздельно (сертификат и ключ) и сразу пригодны для подписывателя. Каталог вывода меняется через `OUT=...` (по умолчанию `data/certs`).

```bash
make cpcert-build # собрать образ (однократно)
# Положи папку контейнера, например, в data/certs/mycont.000
make cpcert DIR=data/certs/mycont.000 PASS=пароль
# -> data/certs/mycont.cer + data/certs/mycont.key  (имена по названию контейнера)
# или если хотите положить ключ-сертификат в свою директорию
make cpcert DIR=data/certs/mycont.000 PASS=пароль OUT=my
# -> my/mycont.cer + my/mycont.key  (имена по названию контейнера)
```

Если нужен один склеенный PEM, то используйте:

```bash
make cpcert-pem DIR=data/certs/mycont.000 PASS=пароль > out.pem
```

- Если контейнер без пароля, то просто не вводить `PASS=`
- Флаг `-debug` печатает промежуточные значения (pwd_key, расшифровка, маска, EC-точка) в stderr. Полезно для отладки/сверки:
  ```bash
  docker run --rm -v "$PWD/data/mycont.000":/c:ro crypto-amateur-cpcert -debug /c "пароль"
  ```

## Сборка нативного бинарника (если есть Go)

```bash
cd cpcert && go build -mod=vendor -o cpcert .
./cpcert data/mycont.000 пароль > out.pem
```
